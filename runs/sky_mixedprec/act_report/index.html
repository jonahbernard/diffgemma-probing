activation outlier report

magenta/red = mxfp4 (W4A4), green = bf16. Outliers inflate fp4 block scales, so big per-block absmax = more quantization error.
What am I looking at?
This report dissects the activations that feed each GEMM (matrix multiply) inside DiffusionGemma, comparing the BF16 model against its MXFP4 (4-bit weight + 4-bit activation) twin on the same prompt. fp4 represents numbers in small blocks that share one scale, so a single huge value (an 'outlier') forces a coarse scale on its whole block and loses precision -- these pages are about finding those outliers: which channels (hidden dims), which tokens (canvas positions), at which layer and denoising step, and how much bigger they are under mxfp4. The 'runs' table below shows how many records each model produced; note the two models took a different number of denoising steps for the same prompt. The paper:* pages reproduce the canonical activation charts from the quantization literature (SmoothQuant channel x layer surface, AWQ/LLM.int8 salient-channel spectrum + heavy-tail histogram, SpinQuant kurtosis-per-layer, Outlier-Suppression+ per-channel ranges), using bf16-vs-mxfp4 in place of their before/after-rotation panels. Start with the paper:* pages or the heatmaps for the big picture, then drill into the channels/tokens pages, and use outlier rankings to jump to the worst spots. magenta/red = mxfp4 (W4A4), green = bf16. Outliers inflate fp4 block scales, so big per-block absmax = more quantization error.
